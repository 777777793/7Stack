## 一、实战使用手册 · 章节导航

### 第 1 章：工程搭建
- **1.1 父子模块结构设计** — Maven 多模块工程规范，common/api/service 分层约定
- **1.2 Spring Cloud 版本选型** — Spring Boot / Spring Cloud / Spring Cloud Alibaba 版本对应表及 2024 推荐组合
- **1.3 公共依赖管理** — `dependencyManagement` 统一版本，避免 JAR 冲突

### 第 2 章：服务注册与发现（Nacos）
- **2.1 Nacos 快速接入** — 三步接入：引依赖→配地址→加注解，含 Docker 一键启动命令
- **2.2 服务分组与命名空间** — 多环境隔离方案，dev/test/prod 切换最佳实践
- **2.3 健康检查机制** — 临时实例 vs 持久实例，心跳检测参数调优
- **2.4 常见问题排查** — 注册失败、服务列表为空、心跳超时的排查路径

### 第 3 章：服务间调用（OpenFeign）
- **3.1 OpenFeign 快速接入** — 接口声明式调用，`@FeignClient` 核心注解详解
- **3.2 负载均衡配置** — Spring Cloud LoadBalancer 替换 Ribbon，权重/轮询策略切换
- **3.3 超时与重试配置** — connectTimeout / readTimeout 企业级推荐值，重试陷阱
- **3.4 Feign 日志与拦截器** — 统一传递 Token、TraceId 的请求拦截器写法
- **3.5 常见报错排查** — `feign.FeignException: 404` 等高频报错解法

### 第 4 章：API 网关（Spring Cloud Gateway）
- **4.1 网关快速接入** — 路由配置、断言（Predicate）、过滤器（Filter）三件套
- **4.2 统一鉴权过滤器** — GlobalFilter 实现 Token 校验，登录白名单配置
- **4.3 限流配置** — RequestRateLimiter + Redis 令牌桶，防刷接口
- **4.4 跨域处理** — CorsWebFilter 全局跨域配置（避开 Spring MVC CORS 冲突）
- **4.5 网关常见陷阱** — 与 Spring MVC 不兼容、WebFlux 响应式编程注意点

### 第 5 章：熔断限流（Sentinel）
- **5.1 Sentinel 快速接入** — Dashboard + 客户端接入，规则持久化到 Nacos
- **5.2 流量控制规则** — QPS 限流、线程数限流，预热（Warm Up）与排队等待模式
- **5.3 熔断降级规则** — 慢调用比例 / 异常比例 / 异常数三种策略选型
- **5.4 @SentinelResource 注解** — blockHandler / fallback 区别，降级兜底写法
- **5.5 与 OpenFeign 整合** — Feign 调用自动熔断，降级返回兜底对象

### 第 6 章：配置中心（Nacos Config）
- **6.1 配置中心接入** — bootstrap.yml 配置，DataId 命名规范
- **6.2 配置动态刷新** — `@RefreshScope` 的正确用法与失效场景
- **6.3 配置共享与继承** — shared-configs / extension-configs 多配置文件加载顺序
- **6.4 敏感配置处理** — 数据库密码等敏感项加密方案（Jasypt 集成）

### 第 7 章：链路追踪（Sleuth + Zipkin）
- **7.1 快速接入** — 引依赖自动生效，TraceId / SpanId 注入日志
- **7.2 Zipkin 可视化** — Docker 启动 Zipkin，采样率配置（生产建议 0.1）
- **7.3 跨服务追踪原理** — HTTP Header 传递 TraceId 的机制说明
- **7.4 与日志框架整合** — MDC 自动注入，ELK 日志聚合时通过 TraceId 关联调用链

## 二、技术深度精讲 · 章节导航

### 第 1 章：微服务 vs 单体架构
- **1.1 单体架构的真实痛点** — 不是"不好"，是规模大了之后哪里撑不住
- **1.2 微服务的代价清单** — 分布式系统八大谬误，微服务引入的复杂度
- **1.3 拆分时机与粒度判断** — 什么时候该拆，按业务域拆还是按 DDD 拆

### 第 2 章：服务注册与发现原理
- **2.1 为什么需要注册中心** — 硬编码 IP 的问题，动态服务发现的必要性
- **2.2 Nacos 注册中心架构** — Raft 协议、AP/CP 模式切换，集群数据同步原理
- **2.3 心跳检测与服务剔除** — 临时实例 5s 心跳、15s 不健康、30s 剔除的状态机
- **2.4 客户端缓存与推送** — 本地缓存 + UDP 推送，网络分区时的自我保护

### 第 3 章：负载均衡算法深度解析
- **3.1 常见算法对比** — 轮询 / 加权轮询 / 随机 / 一致性哈希，各自适用场景
- **3.2 一致性哈希详解** — 虚拟节点解决数据倾斜，哈希环的工作原理
- **3.3 客户端 vs 服务端负载均衡** — Ribbon/LoadBalancer vs Nginx/LVS 的本质区别

### 第 4 章：分布式事务（Seata）
- **4.1 分布式事务的根本问题** — 两阶段提交的悲剧，为什么 XA 在微服务里用不了
- **4.2 Seata AT 模式原理** — undolog 回滚日志、全局锁、二阶段异步提交
- **4.3 Seata TCC 模式** — try/confirm/cancel 三阶段，空回滚与幂等性处理
- **4.4 Saga 与消息最终一致性** — 长事务场景选 Saga，可靠消息 + 幂等消费兜底

### 第 5 章：熔断降级底层机制
- **5.1 为什么需要熔断** — 雪崩效应的形成过程，一个慢服务拖垮全链路
- **5.2 熔断器状态机** — Closed / Open / Half-Open 三态转换，滑动窗口计数
- **5.3 Sentinel vs Hystrix** — 线程隔离 vs 信号量隔离，资源消耗对比
- **5.4 限流算法详解** — 令牌桶 vs 漏桶 vs 计数器，Sentinel 滑动时间窗实现

### 第 6 章：服务网格（Service Mesh）概览
- **6.1 Service Mesh 解决什么问题** — 把治理逻辑从业务代码里剥离出去
- **6.2 Istio 核心架构** — Envoy Sidecar、控制平面（Pilot/Mixer）分工
- **6.3 与 Spring Cloud 的关系** — 不是替代，是不同层面的治理方案
